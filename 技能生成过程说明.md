# 职教资讯技能 — 生成过程说明

## 概述

本技能从零开始，通过多轮对话迭代构建了一个完整的职业教育新闻智能检索与报告生成系统。以下记录完整的演进过程。

---

## 阶段一：建立数据基础

### 输入
用户提供了 `职教AI知识库-信息源网址与栏目映射.md`，内含 17 类信息源的网址与栏目映射表。

### 动作
- 读取并解析 17 类信息源的结构化数据
- 分为三大板块：国内官方（5源）、国际前沿（7源）、实践案例（5源）
- 识别出快速检索索引表（按意图匹配信息源）

### 产出
- 技能文件 `职教资讯.md` v1.0，包含 17 类信息源目录和基本检索流程

---

## 阶段二：首次检索验证（发现问题）

### 动作
首次调用技能检索"最近一月国内职教政策"，对 4 个源执行 `site:` 搜索。

### 发现的问题

| 问题 | 严重度 | 说明 |
|------|:--:|------|
| **cvae.com.cn 无搜索结果** | 高 | 搜索引擎不索引该站 |
| **civte.edu.cn 无搜索结果** | 高 | 同上 |
| **tech.net.cn 子域名死链** | 高 | `old.tech.net.cn` / `www2.tech.net.cn` 返回 000 |
| **Word 链接为纯文本** | 中 | URL 不可点击跳转 |
| **Word 字体不一致** | 中 | 使用了纯西文字体 Arial |

## 阶段三：修复与迭代

### 3.1 URL 校验机制

**问题**：搜索引擎返回的 `old.tech.net.cn` 链接全部 404。

**根因**：搜索引擎索引了已废弃的子域名。

**解决方案**：
- 新增 Step 3.5 URL 校验步骤
- 用 `curl` 逐条验证 HTTP 状态码
- 建立 `URL_FIX_MAP` 死域名映射表
- 固化到技能文件中

```python
URL_FIX_MAP = {
    "old.tech.net.cn": "www.tech.net.cn",
    "www2.tech.net.cn": "www.tech.net.cn",
}
```

### 3.2 Word 超链接修复

**问题**：URL 以纯文本输出，无法点击。

**解决方案**：
- 使用 `python-docx` 的 `OxmlElement('w:hyperlink')` 创建可点击超链接
- 设置蓝色 (#0563C1) + 下划线样式
- 显示文字为「🔗 查看原文」

### 3.3 中文字体修复

**问题**：`font.name = 'Arial'` 是纯西文字体，中文回退到系统默认字体，视觉不一致。

**根因**：`python-docx` 不原生支持东亚字体设置。

**解决方案**：
- 新增 `set_run_font()` 辅助函数
- 通过 `w:rFonts` XML 元素同时设置：
  - `w:ascii` = Calibri（西文）
  - `w:hAnsi` = Calibri（西文高 ANSI）
  - `w:eastAsia` = 微软雅黑（中文）
- 所有 `add_run()` 调用统一替换为 `set_run_font()`

### 3.4 日期过滤机制

**问题**：搜索引擎 `site:2026年6月` 返回全月结果，"最近一周"报告混入了 6 月初旧闻。

**解决方案**：
- 在 Step 1 中新增"日期严格过滤规则"
- 搜索结果逐条检查日期，剔除超出 7 天窗口的条目
- 不足时标注「本周该源无新资讯」而非用旧闻填充

## 阶段四：扩充信息源覆盖

### 4.1 发现六校遗漏

**问题**：信息源 #16（6 所重点院校）从未被单独搜索，院校动态仅通过 tech.net.cn 转载间接获取。

**解决方案**：
- 新增 Step 2.5"必检信息源清单"
- 六校逐站列出搜索规则，涉及院校动态时必须全部单独 `site:` 搜索
- 更新意图匹配表，六校从简写改为逐站列出 + "全部必搜"标注

### 4.2 site: 搜索失效 → 中文校名补搜

**问题**：6 校中 5 校的 `site:` 搜索返回 0 结果（搜索引擎不索引大学官网）。

**解决方案**：
- 改用中文校名直接搜索（如"金华职业技术大学 新闻"）
- 成功率从 17%（1/6）提升到 100%（6/6）
- 固化到技能中

### 4.3 扩充到 14 类可检索源

**本轮新增**：
- **eol.cn**（中国教育在线）— `site:` 搜索可用，6 条/周
- **世校赛**（世界职业院校技能大赛）— 中文名搜索可用，6 条/周
- **央视网/光明网** — 中文名搜索可用
- **ndrc.gov.cn**（发改委）— `site:` 搜索可用但更新频率低

### 4.4 攻坚彻底失败源

| 信息源 | 尝试手段 | 结果 |
|------|------|------|
| **cvae.com.cn** | `site:` 搜索 + WebFetch + curl + 模拟浏览器 UA + API 探测 | 🔴 Vue SPA，内容由 JS 动态渲染，curl 无法获取。需 Puppeteer/Playwright 无头浏览器 |
| **civte.edu.cn** | `site:` 搜索 + WebFetch + curl 直爬 | 🟡 curl 可获取 HTML 内容，但更新频率低（最新 06.01） |

cvae.com.cn 的根因分析：
```
首页返回: <div id="app"></div>  ← Vue SPA 空壳
JS 文件: app.8e86f51a.js       ← 内容由 JS 异步加载
API 端点: 全部不存在            ← 无后端 API 可调用
curl + 浏览器 UA: HTTP 200     ← 但返回的是空壳 HTML
→ 结论: 需要无头浏览器渲染 JavaScript
```

## 阶段五：双输出模式

### 用户需求
- "速览" → 简洁表格式，按信息源分组，每条一句话
- "按主题" → 深度章节式，按主题归类，完整摘要段

### 解决方案
在技能 Step 4 中定义两种输出模式：

| 模式 | 触发词 | 格式 | 摘要长度 |
|------|------|------|:--:|
| A. 速览 | "速览""逐条""简单列" | 按信息源分表格 | ≤40字 |
| B. 主题 | "按主题""分类汇总" | 按主题分章节 | 100-200字 |

生成脚本也对应拆分为两个独立文件：
- `generate_quickview_word.py` — 速览模式
- `generate_thematic_word.py` — 主题模式

## 最终技能架构

```
用户输入
  ↓
Step 1: 确认时间范围（周/月）+ 日期严格过滤
  ↓
Step 2: 意图匹配 → 确定优先检索信息源
  ↓
Step 2.5: 必检源清单检查（六校等强制搜索）
  ↓
Step 3: 并行 site: 搜索 + 中文名补搜 + curl 直爬
  ↓
Step 3.5: URL 校验 + 死域名修复
  ↓
Step 4: 模式选择 → 速览(A) 或 主题(B)
  ↓
Step 5: 生成 Word/HTML（可点击链接 + 双字体）
```

## 核心教训

1. **搜索引擎 `site:` 不可靠**：大学官网索引率极低，需中文校名补搜
2. **SPA 网站无解**：Vue/React 渲染的网站 curl 只能拿到空壳
3. **URL 必须校验**：搜索引擎返回的链接中包含大量已废弃的子域名
4. **日期必须过滤**：`site:2026年6月` 返回全月数据，需手动剔除窗口外条目
5. **Word 字体需双设**：`python-docx` 不原生支持东亚字体，需通过 XML 操作
